# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Лендинг-портфолио «Цифровой мастер»

> Главный документ проекта
> Версия: 2.0 | Дата: март 2026
> Обновлено: Split-Hero концепция + AI-чатбот

---

## 1. Цель сайта

**Главная цель:** Получить заявку на разработку от потенциального клиента.

**Конкретные метрики успеха:**
- Посетитель оставляет заявку через форму или пишет в Telegram
- Целевая конверсия: 3-5% (из посетителя в заявку)
- Время на сайте: > 60 секунд
- Показатель отказов: < 40%

**Одностраничный лендинг (SPA)** — вся информация на одной странице, плавный скролл между секциями.

## 2. Целевая аудитория

### Первичная ЦА: владельцы малого и среднего бизнеса

| Параметр | Описание |
|----------|----------|
| Возраст | 28-50 лет |
| Доход | Средний и выше |
| Боль | Нет сайта / сайт не приносит заявок / нет автоматизации / хотят внедрить AI |
| Страх | Срыв сроков, непрозрачная работа, потеря денег |
| Желание | Получить работающий цифровой продукт под ключ, быстро, с понятным процессом |
| Техническая грамотность | Низкая-средняя. Не знает, что такое React, API, CI/CD |

### Вторичная ЦА: стартапы и предприниматели
- Нужен MVP быстро
- Ищут одного исполнителя «под ключ» вместо студии
- Ценят скорость и гибкость

### Язык коммуникации
- Русский, простой, без жаргона
- Вместо «Разработка Telegram Bot API» → «Сделаю бота, который запишет клиентов»
- Вместо «Fullstack-разработка SPA» → «Создам сайт, который приносит заявки»

### Стилистические правила

- Тёмная тема как основная и единственная
- Скруглённые углы на карточках: `border-radius: 16-20px`
- Glassmorphism на навигации и карточках: `backdrop-filter: blur(20px)`
- Grain-текстура поверх всего сайта (opacity: 0.03-0.05)
- Aurora-блобы (размытые цветные пятна) на фоне hero и CTA-секций
- Большие отступы между секциями: `padding: 100-120px 0` (desktop)
- Максимальная ширина контента: `max-width: 1200px`

**Реализация слайдера (JS):**

```javascript
// Принцип работы:
// - Два блока с overflow: hidden, вложенные в контейнер
// - clip-path или width% управляется позицией мыши/тача
// - Одно фото/изображение под обоими слоями
// - CSS: левый блок clip-path: inset(0 50% 0 0)
//         правый блок clip-path: inset(0 0 0 50%)
// - JS: при mousemove/touchmove обновляет % разделения
// - Плавность: GSAP lerp или CSS transition

const slider = document.querySelector('.split-slider');
const leftSide = document.querySelector('.split-left');
const rightSide = document.querySelector('.split-right');

slider.addEventListener('mousemove', (e) => {
  const rect = slider.getBoundingClientRect();
  const x = ((e.clientX - rect.left) / rect.width) * 100;
  const clampedX = Math.max(20, Math.min(80, x)); // ограничение 20-80%
  leftSide.style.clipPath = `inset(0 ${100 - clampedX}% 0 0)`;
  rightSide.style.clipPath = `inset(0 0 0 ${clampedX}%)`;
  dividerLine.style.left = `${clampedX}%`;
});

// Touch-поддержка для мобильных
slider.addEventListener('touchmove', (e) => { /* аналогично */ });
```

**Двухсторонняя стилистика всего лендинга:**

Split-концепция задаёт визуальный лейтмотив для всего сайта:
- В секции **Услуги**: карточки дизайна — тёплый gradient border, карточки разработки — холодный neon border
- В секции **Процесс**: левая часть шагов (анализ, дизайн) — тёплые тона, правая (разработка, запуск) — холодные
- В секции **Обо мне**: фото с лёгким split-эффектом (половина тёплая, половина холодная)
- Общий паттерн: дизайн = тёплые тона (indigo/fuchsia), код = холодные (cyan/green)

**Адаптив (мобильный):**
- Слайдер работает по touchmove
- Split-зона уменьшается до 60% высоты экрана
- Текст и CTA переносятся под split-зону
- Ручка слайдера увеличивается для удобства тача (60px)

**Анимации:**
- При загрузке: слайдер автоматически «проезжает» от 30% до 70% и обратно к 50% (демонстрация интерактивности, 2 сек)
- Левая сторона: плавающие дизайн-элементы (CSS float animation)
- Правая сторона: бегущий код-текст (CSS marquee vertical), мерцание терминала
- Разделитель: пульсирующий glow
- Надзаголовок: typewriter с мигающим курсором
- H1: посимвольное появление (GSAP SplitText), gradient-text
- Подзаголовок: fade-in снизу, задержка 0.5s
- Счётчики: count-up (0 → 50+)
- CTA: fade-in + магнитный эффект, glow на hover
- Scroll indicator: анимированная стрелка

**Задача:** Снять страх неопределённости, показать прозрачность.

**Контент — 5 шагов (горизонтальный таймлайн):**

```

[01]                  [02]                 [03]
Бриф и анализ         Проектирование       Разработка
1-2 дня               3-5 дней             2-4 недели

Обсуждаем задачу,     Прототип,            Пишу код, показываю
изучаю ваш бизнес,    структура,           промежуточные
```

| Stagger появление | Карточки болей, услуг | CSS/GSAP stagger |
| 3D tilt карточки | Услуги | JS mousemove |

### Средний приоритет (добавить после MVP):

| Анимация | Где | Технология |
|----------|-----|-----------|
| Кастомный курсор | Весь сайт (только desktop) | JS + CSS |
| Grain-overlay | Весь сайт | SVG texture |
| Scroll progress bar | Верх страницы | CSS animation-timeline |
| Карусель отзывов | Отзывы | JS/CSS |
| Параллакс-слои | Hero фон | GSAP ScrollTrigger |

### Правила анимаций:
- Все анимации отключаются для `prefers-reduced-motion: reduce`
- На мобильных: упрощённые анимации (без 3D tilt, без кастомного курсора, без горизонтального скролла)
- Загрузка GSAP — lazy, после первого рендера
- Максимальная длительность одной анимации: 0.8s (reveal), 1.5s (hero)
- Easing: `ease-out` для появлений, `elastic.out` для интеракций
